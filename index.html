Git

Base

  • git config --global user.name blabla - встановити name користувача.
  • git config --global user.email blabla - встановити email користувача.
  • git init - створити новий репозиторій.
  • git status - покаже нам, які файли не були добавлені в список для наступної фіксації.
  • git add - додати всі змінені файли (до stage).
  • git commit - створити новий коміт.
  • git log - показати історію комітів
  • git checkout - повернутись до останнього коміту.
  • git branch - показати всі гілки.
  • git push - вивантажує вміст локального репозиторію у віддаленний.
  • git pull - завантажує вміст віддаленного репозиторію.
  • git merge - з'єднати 2 гілки в одну.
  • git clone - клонує віддалений репозиторій.

.gitignore - це простий текстовий файл, в кожному рядку якого міститься шаблон файлу або каталогу, який необхідно проігнорувати.

Advanced

  • git revert - створює "антикомміт" для вказаного комміту, таким чином відміняя зміни, внесені в ньому.
  • git reset - скасовує внесені в комміті зміни і додає новий комміт з отриманим вмістом.
  • git rebase - спосіб перенести зміни з одної гілки в іншу.
  • git cherry-pick - використовується для перенесення окремих коммітів з одного місця репозиторію в інше.
  • git stash - призначена для того, щоб помістити поточні зміни, які ми виконали у файлах, в окреме сховище, та повернути файли до вихідного стану.
